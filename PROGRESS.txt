[✓] Android 7.0+ compatible
[✓] GitHub Actions auto-build
[✓] Green Kali/Linux aesthetic

CURRENT CAPABILITIES:
- Execute shell commands from /system/bin
- Use pipes: ps | grep
- Use redirects: echo > /tmp/file
- Access environment: HOME, PATH, TERM
- Navigate filesystem: ls, cd, pwd
- View system info: uname, ps, cat /proc/*
- Full keyboard support with history

TESTED COMMANDS:
ls, cd, pwd, echo, cat, grep, ps, uname, date, touch, rm, mkdir

LIMITATIONS (by design):
- No root (not needed)
- No additional packages yet (Phase 2)
- Cannot modify /system (sandbox)
- Limited to /system/bin binaries

                      HOW TO TEST KERNUX
=================================================================================

QUICK START:
1. Install APK on device
   adb install kernux-debug-apk.apk

2. Launch app
   Should see black screen with green prompt

3. Run basic commands:
   pwd                     # Current directory
   ls -la                  # List files with colors
   echo "Hello Kernux"     # Print text
   ps                      # List processes
   uname -a                # System info

4. Test advanced:
   ps | grep shell         # Test pipes
   echo test > /tmp/t      # Test redirects
   UP arrow                # Test history

For complete testing guide, see TESTING.md

                        PRACTICE COMMANDS

BASIC COMMANDS (start here):
pwd                        # Show current directory

ls -la                     # List with details
cd /tmp                    # Change directory

touch /tmp/test.txt        # Create file

cat /tmp/test.txt          # Read file
rm /tmp/test.txt           # Delete file

SYSTEM INFORMATION:
uname -a                   # Kernel version
ps                         # List processes
date                       # Show date/time
cat /proc/cpuinfo | head   # CPU information
echo $HOME                 # Home directory
echo $PATH                 # Search path

TEXT PROCESSING:
grep "text" file.txt       # Search in file
sed "s/old/new/" file.txt  # Replace text
awk "{print $1}" file.txt  # Print fields
wc -l file.txt             # Count lines

PIPES & REDIRECTS:
ps | grep shell            # Filter processes
ls | wc -l                 # Count files
echo "test" > file.txt     # Write to file
echo "more" >> file.txt    # Append to file
cat file.txt | grep word   # Pipe through grep

VARIABLES:
export MYVAR="value"       # Set variable
echo $MYVAR                # Show variable
echo $HOME                 # Home directory
echo $PATH                 # Search path

Goal: Create and manage files
1. touch /tmp/myfile.txt   # Create file
2. echo "Line 1" > /tmp/myfile.txt  # Write
3. echo "Line 2" >> /tmp/myfile.txt # Append
4. cat /tmp/myfile.txt     # Read file
5. rm /tmp/myfile.txt      # Delete file
Result: Comfortable with file operations

EXERCISE 3: System Information
Goal: Understand system environment
1. uname -a                # Kernel info
2. ps | head -5            # Show processes
3. echo $HOME              # Home directory
4. cat /proc/cpuinfo | head-5  # CPU info
Result: Know system details

EXERCISE 4: Pipes & Filters
Goal: Learn data filtering
1. ps                      # All processes
2. ps | grep init          # Filter for init
3. ps | grep sh            # Filter for shells
4. ls /tmp | wc -l         # Count /tmp files
Result: Comfortable with pipes

EXERCISE 5: Advanced

Result: Handle advanced commands
